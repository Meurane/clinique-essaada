# Briefs visuels — Clinique ESSAADA

> Direction validée 2026-04-20 : **« Clean clinique européenne + touche de chaleur »**
>
> Référence visuelle globale : Doctolib, Livi, Alan, Qare, Ramsay Santé — propreté nordique, photos posées nettes — avec une chaleur méditerranéenne subtile (textiles crème, lumière dorée douce, visages algériens / diaspora).
>
> À utiliser directement comme prompts pour générateurs IA (Midjourney v6, Flux 1.1 Pro, DALL-E 3) **ou** comme brief pour photographe / illustrateur humain.

---

## 1 · Direction artistique globale (à répéter en tête de tous les prompts)

### Mood
Professionnel médical européen contemporain. Propre sans être stérile. Chaleureux sans être familial. Rassurant sans être niais. Registre éditorial — comme si c'était le site d'une clinique haut de gamme en Île-de-France ou Catalogne, avec des visages d'Afrique du Nord.

### Palette (à nommer explicitement dans chaque prompt)
- **Dominantes** : blanc cassé, crème, beige lin (`#FBF8F2`, `#F2ECE0`)
- **Accent principal** : teal profond méditerranéen (`#0A475D` — présent sur blouses, signalétique, mobiliers)
- **Accents secondaires** : sable lin (`#E3D8C3`), vert émeraude très parcimonieux (`#059669` — badges, indicateurs)
- **Interdits** : rouges saturés (sauf rare sang médical), jaunes-orangés parasites, bleus électriques, violets, fluos

### Lumière
- **Naturelle exclusivement**. Fenêtre latérale douce, ciel couvert ou golden hour discret (fin matinée 10h-11h).
- **Jamais** : flash direct, néon clinique vert, contre-jour dramatique, lumière rasante style cinéma.

### Composition & cadrage
- Règle des tiers. Focus net sur sujet. Arrière-plan propre (mur clinique, rideau lin, plante discrète).
- Pas de grand-angle distordu, pas de plongée/contre-plongée dramatique.
- Espace négatif généreux — laisse respirer.

### Casting (pour photos avec personnes)
- Soignants 30-55 ans, majoritairement algériens / maghrébins. Femmes et hommes équilibré.
- Patients 45-75 ans, visages variés (local + diaspora 2e génération). Dignité, pas misérabilisme.
- Blouses blanches impeccables avec liseré ou badge ESSAADA en teal. Pas de tenue civile.
- Sourires naturels et retenus — pas sourire dentifrice publicitaire.

### Post-prod / LUT
- Légère désaturation rouges (−8 %).
- Boost teal dans les ombres (très léger, +3 %).
- Peau naturelle, pas de lissage exagéré.
- Grain argentique 1-2 % (invisible à l'œil, texture subtile).

### Interdits absolus
- Stock photo clichés (soignants blancs mains jointes vers ciel, sourire à 32 dents).
- Matériel fictif ou ancien.
- Ambiance "hôpital public surchargé" — ESSAADA est un centre privé moderne.
- Patients en position vulnérable non-consentie (visage apparent sans consentement explicite).

## 3 · Prompts illustrations médicales (6 schémas)

> Pour un illustrateur vectoriel (Adobe Illustrator, Figma) ou AI Flux / Midjourney niche-style.
> Toutes les illustrations doivent être **SVG exportables**, fond transparent, palette stricte ESSAADA, traits 2px arrondis, style **cohérent entre toutes les illustrations** (c'est une série, pas des one-shots).

### Direction artistique illustrations

**Style** : Line-drawing éditorial moderne. Traits teal (`#0A475D`) nets, remplissages subtils en sand (`#F2ECE0`) ou teal clair (`#D3E7EE`). Jamais de rouge sanguin, jamais de détails anatomiques effrayants. Simplicité pédagogique.

**Référence visuelle** :
- *Medium Health illustrations*
- *Hopscotch editorial medical*
- *Pitch Medical* (le studio de brand illustration pour scale-ups santé)
- *Airbnb Lottie illustrations* (pour la douceur)
- **NON** : illustrations médicales textbook Netter, manuels scolaires clinical, dessins techniques Planche anatomique.

**Palette stricte** :
- Trait principal : `#0A475D` (primary-700)
- Fill accent doux : `#D3E7EE` (primary-100)
- Fill secondaire chaleur : `#F2ECE0` (sand-100)
- Highlights ponctuels : `#0F5E79` (primary-600)
- Neutre texte : `#44403C` (neutral-700)
- **Jamais** : rouge, bleu électrique, gris clinique, couleurs saturées

**Format** : SVG vectoriel, viewBox carré 600×600 ou rectangulaire selon composition, fond transparent, compatibles mode sombre (traits suffisamment épais pour rester lisibles sur fond teal-700).

## 4 · Notes techniques (intégration & specs)

### Formats de livraison
| Type | Format natif | Format web | Remarques |
|---|---|---|---|
| Photos | RAW + JPEG 100 % | WebP + AVIF (via `next/image`) | Min 2400×3000 pour portrait, 3000×2000 pour paysage |
| Illustrations | AI vectoriel + SVG | SVG inline ou `next/image` | Nettoyer les IDs, compresser via SVGO |

### Dimensions attendues par composant
- **PhotoHero** : 5:4 ou 4:5, 2400 px côté court min
- **PortraitCard** : 4:5, 1600 px hauteur min
- **PhotoGrid** : variable, optimal 1600 px côté court
- **PhotoCallout** : 1:1, 1200 px min
- **PhotoStory** : 4:5 ou 5:4, 2000 px min

### Naming convention
```
/public/images/
  ├── team/
  │   ├── dr-nom-nephrologue-01.webp
  │   └── ...
  ├── scenes/
  │   ├── seance-branchement-01.webp
  │   └── ...
  ├── lieux/
  │   ├── salle-dialyse-wide-01.webp
  │   ├── accueil-reception-01.webp
  │   └── ...
  ├── details/
  │   ├── mains-carnet-dialyse-01.webp
  │   └── ...
  └── illustrations/
      ├── rein-anatomie.svg
      ├── stades-irc-timeline.svg
      ├── dfg-filtration.svg
      ├── fistule-acces.svg
      └── parcours-patient.svg
```

### Consentement RGPD (obligatoire pour toute photo avec personne identifiable)
- Formulaire écrit signé avant prise de vue
- Droit de retrait à tout moment
- Usage limité au site + supports officiels (pas de revente)
- Mineurs : double consentement (tuteur légal)
- Modèle de formulaire à préparer avec juriste DSP Sidi Bel Abbès

### Checklist avant publication
- [ ] Traitement LUT uniforme appliqué (cohérence inter-photos)
- [ ] Export WebP 85 % + AVIF 80 % qualité
- [ ] Alt text descriptif rédigé pour chaque photo
- [ ] Consentement archivé (si personne)
- [ ] Résolution responsive : 400w, 800w, 1200w, 1600w, 2400w
- [ ] Blur placeholder généré (`plaiceholder` npm)

## 5 · Phase de production recommandée

| Étape | Délai | Livrable |
|---|---|---|
| **P0 — Maintenant** | semaine 1 | Site live avec PhotoPlaceholder propres (✅ fait) |
| **P1 — Illustrations** | 2-3 semaines | 6 SVG livrés par illustrateur · pose immédiate · peu de dépendance opérationnelle |
| **P2 — Shooting** | 1-2 jours cliniques | 10 scènes photos via photographe documentaire local (Alger / Oran) |
| **P3 — Post-prod** | 1 semaine | LUT + compression + variants responsive |
| **P4 — Intégration** | 2-3 jours dev | Remplacer `src=undefined` par `src="/images/..."` dans composants |

Les composants `PhotoHero`, `PhotoStory`, `PortraitCard`, `PhotoGrid`, `PhotoCallout` acceptent tous la prop `src?: string` — **zéro refactor nécessaire** quand les photos arriveront. On passe de placeholder à photo réelle via un simple ajout de prop.
